## 2024-04-17 - Prevent Exposure of Profiling Endpoints via `http.DefaultServeMux`
**Vulnerability:** The project previously imported `net/http/pprof` and `expvar` in main entry points (e.g. `cmd/tesseract/posix/main.go`). This inadvertently registers `/debug/pprof/*` and `/debug/vars` handlers on the global `http.DefaultServeMux`. If an application relies on `http.DefaultServeMux` (or unintentionally exposes it), this could expose sensitive internal metrics, memory states, and profiling data (CWE-200), aiding attackers in reconnaissance and exploitation.
**Learning:** In Go, the `_ "net/http/pprof"` and `_ "expvar"` imports have init() side-effects that register routes globally on `http.DefaultServeMux`. Cloud entry points with custom servers might still inadvertently expose this if a component internally registers with the default mux or if it's explicitly routed.
**Prevention:** Never use blank imports (`_ "net/http/pprof"`, `_ "expvar"`) in production binaries or public-facing HTTP handlers. If profiling is needed, register handlers manually to an explicitly secured (e.g., internal-only, authenticated) custom `http.ServeMux` instead of the global default.
