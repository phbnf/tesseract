## 2026-07-03 - Exposing Debug Endpoints on Public Interfaces (CWE-200)
**Vulnerability:** Anonymous imports of `net/http/pprof` and `expvar` automatically register their handlers (`/debug/pprof/*` and `/debug/vars`) on the global `http.DefaultServeMux`. If an application relies on `http.DefaultServeMux` for its public-facing server (e.g., via `http.Handle` or `http.ListenAndServe(addr, nil)`), these internal diagnostic endpoints become publicly exposed.
**Learning:** These endpoints can leak sensitive information such as server internals, performance metrics, and database state (e.g., BadgerDB metrics via `expvar`), which can aid attackers in reconnaissance and exploiting other vulnerabilities.
**Prevention:** Avoid anonymous imports of `net/http/pprof` and `expvar` in binaries intended for production use, especially when they expose public HTTP endpoints. Use custom `http.ServeMux` instances for public endpoints rather than relying on the global `http.DefaultServeMux`, and ensure diagnostic endpoints are only registered on internal administrative networks or disabled entirely in production.
