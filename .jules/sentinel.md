## 2026-05-07 - Profiling Endpoint Exposure via Blank Imports
**Vulnerability:** The codebase imported `net/http/pprof` and `expvar` using the blank identifier (`_`) in public-facing binaries (`cmd/tesseract/posix/main.go` and `cmd/tesseract/gcp/main.go`). This automatically registered profiling and metric handlers on `http.DefaultServeMux`, inadvertently exposing `/debug/pprof/*` and `/debug/vars` without authentication, leading to a CWE-200 (Information Exposure) vulnerability.
**Learning:** Even if a custom router or mux is used for the primary application logic, relying on `http.ListenAndServe` or components that fall back to `http.DefaultServeMux` makes these hidden endpoints accessible. Blank imports for these packages are highly dangerous in production entry points.
**Prevention:** Never use blank imports (`_ "net/http/pprof"`, `_ "expvar"`) in application entry points or production binaries. If profiling or metrics are required, explicitly register them on a dedicated, internal-only router or HTTP server bound to a localhost/private interface, separate from the public-facing API.
