## 2025-05-16 - Prevent Unintended Exposure of Profiling and Metrics Endpoints
**Vulnerability:** The HTTP server entry points for the GCP and POSIX cloud personalities (`cmd/tesseract/gcp/main.go` and `cmd/tesseract/posix/main.go`) included anonymous imports of `net/http/pprof` and `expvar`. This implicitly registered their HTTP handlers (`/debug/pprof/*` and `/debug/vars`) onto the global `http.DefaultServeMux`. Since these services use the default mux or multiplexers that could inadvertently expose it, these internal profiling and metrics endpoints were publicly accessible, leading to an Information Exposure vulnerability (CWE-200).
**Learning:** Anonymous imports of packages like `net/http/pprof` and `expvar` automatically attach routes to `http.DefaultServeMux` during `init()`. When developing public-facing HTTP services, relying on the default mux or carelessly exposing it without explicit routing controls and authentication can expose sensitive internal metrics and application state.
**Prevention:** Avoid anonymous imports of `net/http/pprof` and `expvar` in production entry points. If profiling or debug metrics are required, explicitly register them on a separate, internal-only HTTP server (e.g., bound to `localhost` or an admin port) or protect them with strict authentication and authorization middleware.
