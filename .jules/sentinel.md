## 2025-07-06 - Sentinel: [Medium] Remove pprof and expvar to prevent information exposure
**Vulnerability:** The application was anonymously importing `net/http/pprof` and `expvar` in `cmd/tesseract/gcp/main.go` and `cmd/tesseract/posix/main.go`. This automatically registers profiling (`/debug/pprof/*`) and metrics (`/debug/vars`) endpoints on the globally accessible `http.DefaultServeMux`. If `DefaultServeMux` is unintentionally exposed (even on a port separate from the main API if using multiple listeners, or if the main API switches to using it), this could lead to a CWE-200 (Exposure of Sensitive Information to an Unauthorized Actor) vulnerability, leaking application metrics, heap dumps, and goroutine profiles.
**Learning:** Default Go HTTP multiplexers (`http.DefaultServeMux`) implicitly register handlers from imported packages like `pprof` and `expvar`. This behavior is unsafe for public-facing cloud entry points unless explicitly authenticated and secured.
**Prevention:** Avoid using anonymous imports for `net/http/pprof` and `expvar` in production cloud application entry points. If profiling or metrics are necessary, explicitly bind them to a separate internal listener or use safer abstractions like OpenTelemetry which do not rely on global side effects.
