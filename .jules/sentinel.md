## 2026-05-21 - Remove default ServeMux profiling and metrics exposure
**Vulnerability:** The application was exposing `net/http/pprof` and `expvar` handlers by anonymously importing them (`_ "net/http/pprof"` and `_ "expvar"`). This implicitly registers sensitive debugging endpoints on `http.DefaultServeMux` (CWE-200), potentially leaking profiling, memory, and internal state data to unauthenticated users if `http.DefaultServeMux` is used or exposed directly by the server.
**Learning:** Anonymous imports of these packages automatically register handlers on the global mux. Since this app runs custom log handlers directly attached to `http.Handle("/", handler)`, `http.DefaultServeMux` is exposed, making these default debugging paths (`/debug/pprof/` and `/debug/vars`) accessible publicly.
**Prevention:** Do not import `net/http/pprof` or `expvar` unless intentionally binding them to a separate, private HTTP multiplexer that is strictly authenticated or bound to a private internal network interface.
