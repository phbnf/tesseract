## 2025-02-24 - [HIGH] Fix exposure of profiling and metrics endpoints (CWE-200)
**Vulnerability:** The codebase had anonymous imports of `_ "net/http/pprof"` and `_ "expvar"` in the GCP and POSIX HTTP server entry points. These automatically expose internal profiling endpoints (`/debug/pprof/*`) and metrics (`/debug/vars`) onto the default, globally shared `http.DefaultServeMux`.
**Learning:** Even though custom multiplexers might be used in parts of the application, importing these packages anywhere in the `main` package runs their `init()` functions, which register the debug endpoints on the global `DefaultServeMux`. If `http.ListenAndServe` or `http.Server` are used without explicitly defining a router (or if the default mux is inadvertently exposed), it introduces an Information Exposure (CWE-200) risk, making internal metrics, environment variables, and profiling data accessible to attackers.
**Prevention:** Do not use `_ "net/http/pprof"` or `_ "expvar"` in production binaries unless these endpoints are strictly required and explicitly bound to a secure, private, or authenticated admin port separate from public traffic. Rely on explicitly registered OpenTelemetry metrics and handlers instead.
