## 2026-04-16 - Prevent Inadvertent Profiling and Variables Exposure on DefaultServeMux
**Vulnerability:** The POSIX cloud personality binary exposed sensitive profiling (`/debug/pprof`) and internal metrics (`/debug/vars`) endpoints publicly because it imported `net/http/pprof` and `expvar` for side-effects, which automatically register their handlers on `http.DefaultServeMux`. This leads to a CWE-200 vulnerability.
**Learning:** `http.Server` uses `http.DefaultServeMux` if no handler is explicitly provided, or if `http.Handle` is used. Importing packages like `net/http/pprof` and `expvar` implicitly modifies the default mux, inadvertently leaking internal state to anyone able to access the HTTP server on the given address/port.
**Prevention:** Do not import `net/http/pprof` or `expvar` in public-facing binaries. If profiling or internal metrics are necessary, explicitly bind them to an internal, firewalled `ServeMux` on a separate port or interface, and never to the globally exposed `http.DefaultServeMux`.
