## 2024-06-15 - Exposure of profiling and metrics endpoints (CWE-200)
**Vulnerability:** The `net/http/pprof` and `expvar` packages were being anonymously imported (`_ "net/http/pprof"`, `_ "expvar"`) in the cloud personality binaries (e.g. `cmd/tesseract/gcp/main.go`, `cmd/tesseract/posix/main.go`). This automatically registers profiling (`/debug/pprof/*`) and metrics (`/debug/vars`) handlers on the globally accessible `http.DefaultServeMux`, which gets exposed publicly via the main web server, leaking sensitive internal runtime metrics and memory/CPU profiles to potential attackers.
**Learning:** Anonymous imports of standard library debugging tools automatically attach themselves to the global multiplexer, bypassing any specific router configurations intended to restrict access.
**Prevention:** Avoid anonymous imports of `net/http/pprof` and `expvar`. If profiling or internal metrics are required, attach them explicitly to a dedicated, internal-only router or admin server running on a separate port that is not accessible from the public internet. Use standard OpenTelemetry approaches for metrics/traces that don't rely on `expvar`.
