## 2025-02-14 - Fix CWE-200 profiling endpoint exposure
**Vulnerability:** The `net/http/pprof` and `expvar` packages were imported for side effects (`_ "net/http/pprof"`) in `cmd/tesseract/gcp/main.go` and `cmd/tesseract/posix/main.go`. This automatically registers profiling and metrics endpoints (`/debug/pprof/*` and `/debug/vars`) on the `http.DefaultServeMux`. Since these HTTP server entry points serve public traffic, this inadvertently exposed internal runtime information and metrics, creating a CWE-200 vulnerability.
**Learning:** Importing `net/http/pprof` or `expvar` in binaries that expose an HTTP server using `http.DefaultServeMux` (or handlers derived from it) can unintentionally expose sensitive runtime endpoints to unauthorized users.
**Prevention:** Avoid blanket imports of `net/http/pprof` or `expvar` in public-facing server binaries. If profiling is needed, register these handlers on a separate, internal-only `http.ServeMux` or an authenticated endpoint.
