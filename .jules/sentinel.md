## 2025-04-30 - Prevent default multiplexer exposure of profiling endpoints
**Vulnerability:** The POSIX cloud personality binary implicitly imported `net/http/pprof` and `expvar`, which automatically attached profiling and metrics endpoints to the global `http.DefaultServeMux`. This allows any client capable of reaching the server to query for runtime performance profiles and internal metrics (CWE-200), bypassing security controls and leaking sensitive runtime info.
**Learning:** `import _ "net/http/pprof"` and `import _ "expvar"` modify the global state of the default server multiplexer. When production entry points (such as `cmd/tesseract/posix/main.go`) then use or allow access to the default server multiplexer, these internal debugging endpoints are inadvertently exposed to untrusted traffic.
**Prevention:** Avoid blanket importing `net/http/pprof` and `expvar` in any production binary. If profiling or metrics endpoints are strictly necessary for production, they should be explicitly attached to a dedicated, internal-only multiplexer on a separate port or tightly access-controlled, not left globally available on the public-facing server.
