## 2025-02-27 - Information Exposure via Default HTTP Handlers (CWE-200)
**Vulnerability:** The application blindly imported `net/http/pprof` and `expvar` in the `cmd/tesseract/gcp/main.go` and `cmd/tesseract/posix/main.go` entry points. Because these packages register themselves on `http.DefaultServeMux` by default via their `init()` functions, this exposes application profiling endpoints (`/debug/pprof/*`) and metrics variables (`/debug/vars`) on the public-facing HTTP server if `http.DefaultServeMux` is inadvertently served or exposed by the custom router handler. This is a severe Information Exposure (CWE-200) issue as it leaks process details, memory stats, and BadgerDB metrics.
**Learning:** Default imports in Go standard library often auto-register debug features to global state which developers might not realize when wiring up an `http.Server`.
**Prevention:** Avoid blanket `_ "net/http/pprof"` or `_ "expvar"` imports in public cloud personality binaries. If debug endpoints are needed, bind them to a specifically guarded local-only multiplexer on a separate port rather than exposing them on the primary server multiplexer. Utilize OpenTelemetry for handler observability.
