## 2024-05-30 - Exposure of pprof and expvar endpoints
**Vulnerability:** The `cmd/tesseract/posix/main.go` and `cmd/tesseract/gcp/main.go` binaries inadvertently import `_ "net/http/pprof"` and `_ "expvar"`, which registers these profiling and metrics endpoints globally on `http.DefaultServeMux`. This leads to CWE-200, exposing sensitive information about the application state, memory usage, and performance characteristics to unauthenticated users when the HTTP server starts with `http.Handle("/", logHandler)` and serves traffic directly without filtering these default endpoints.
**Learning:** Anonymous imports of standard library debugging tools like `net/http/pprof` or `expvar` automatically attach their handlers to the default global multiplexer `http.DefaultServeMux`. When servers are exposed to the public internet, these endpoints provide a rich source of internal details to potential attackers.
**Prevention:** Avoid importing `_ "net/http/pprof"` and `_ "expvar"` in production binaries. If metrics or profiling are needed, they should be configured securely, ideally via dedicated, authenticated administrative endpoints or explicitly on a separate, non-public multiplexer, rather than globally.
