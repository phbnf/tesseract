## 2026-06-11 - Global Debug Endpoint Exposure in Cloud Binaries
**Vulnerability:** The AWS/GCP/POSIX cloud application binaries implicitly expose sensitive `/debug/pprof/*` and `/debug/vars` endpoints by including `_ "net/http/pprof"` and `_ "expvar"` imports. These endpoints are bound to the `http.DefaultServeMux`, which may be inadvertently exposed if a server starts on it without tight routing restrictions (CWE-200).
**Learning:** In Go, anonymous imports of `net/http/pprof` or `expvar` automatically register handlers on `http.DefaultServeMux`. When cloud binaries initialize custom servers, the global mux can accidentally serve these endpoints to untrusted networks, leaking profiling data and application metrics.
**Prevention:** Avoid blanket `_ "net/http/pprof"` and `_ "expvar"` imports in public-facing binaries. If debugging is necessary, explicitly register these routes on an internal, authenticated `http.ServeMux` separate from the public internet, or rely on secure observability tools like OpenTelemetry.
