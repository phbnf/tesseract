## 2024-05-18 - Profiling and metrics endpoint exposure via anonymous imports
**Vulnerability:** Profiling (`/debug/pprof`) and metrics (`/debug/vars`) endpoints were automatically exposed on `http.DefaultServeMux` due to anonymous imports of `net/http/pprof` and `expvar` in the `main` packages of cloud personalities. This exposes internal system information (CWE-200), potentially aiding attackers in discovering other vulnerabilities or gathering information about the application's internal state.
**Learning:** Anonymous imports of standard library packages like `net/http/pprof` and `expvar` inherently register HTTP handlers to the globally exposed `http.DefaultServeMux`. When `http.ListenAndServe` or similar server starts up with a `nil` handler, it uses this default mux, making these endpoints public.
**Prevention:** Avoid using `_ "net/http/pprof"` and `_ "expvar"` in production or cloud personality entry points. Use custom HTTP multiplexers (e.g., `http.NewServeMux()`) and explicitly register profiling or metric handlers only on internal or secured administrative endpoints.
