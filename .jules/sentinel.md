## 2024-03-05 - [Fix CWE-200] Mitigated Information Exposure via pprof and expvar
**Vulnerability:** The cloud personalities for GCP (`cmd/tesseract/gcp/main.go`) and POSIX (`cmd/tesseract/posix/main.go`) were inadvertently exposing profiling endpoints (`/debug/pprof/*`) and metrics endpoints (`/debug/vars`) because of anonymous imports of `net/http/pprof` and `expvar`. These libraries automatically register handlers on `http.DefaultServeMux`. If `http.DefaultServeMux` is exposed or custom routers aren't carefully isolated from it, this can leak sensitive application details (CWE-200).
**Learning:** Anonymous imports of standard library debugging tools (`net/http/pprof`, `expvar`) are dangerous in production server code as they have global side effects on the default HTTP multiplexer.
**Prevention:** Avoid `_ "net/http/pprof"` and `_ "expvar"` in production binaries. If profiling is needed, explicitly configure and secure specific endpoints with authentication/authorization rather than relying on the globally accessible `http.DefaultServeMux`.
