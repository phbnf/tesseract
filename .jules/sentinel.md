## 2025-03-21 - Profiling Endpoint Exposure via `net/http/pprof`
**Vulnerability:** The profiling endpoint `net/http/pprof` was inadvertently exposed globally through `http.DefaultServeMux` because `_ "net/http/pprof"` was imported in the GCP and POSIX cloud personality main entry point files. This exposed profiling and runtime data, leading to a CWE-200 (profiling endpoint exposure) vulnerability.
**Learning:** Importing `_ "net/http/pprof"` purely for local debugging can accidentally persist into production code and automatically register handlers on `http.DefaultServeMux`. Even when a custom `http.ServeMux` is primarily used, fallback to the default mux or use of functions like `http.Handle` will expose these endpoints.
**Prevention:** Avoid blanket importing `net/http/pprof` in production-bound code (like main cloud entry points). If profiling is needed, register pprof handlers specifically on an internal or protected HTTP multiplexer instead of the default one, or use OpenTelemetry traces/metrics.
