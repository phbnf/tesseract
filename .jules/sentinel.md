## 2026-05-15 - Remove default HTTP mux profiler exposure (CWE-200)
**Vulnerability:** Anonymous imports of `net/http/pprof` and `expvar` automatically expose sensitive application internals (metrics, memory profiles, CPU profiles) at `/debug/pprof/*` and `/debug/vars` on the global `http.DefaultServeMux`. If an application later binds this mux to a publicly accessible HTTP server, these endpoints leak internal service state and debugging information to external actors, which may aid further exploitation (CWE-200).
**Learning:** Development tools and metric systems that globally register handlers on `http.DefaultServeMux` must be avoided in cloud-facing web server entry points (e.g. `cmd/tesseract/*/main.go`).
**Prevention:** Rely strictly on explicit telemetry mechanisms (e.g., OpenTelemetry) rather than legacy Go profiling/metrics packages in production environments. Never use `_ "net/http/pprof"` or `_ "expvar"` in binaries intended for production server deployments.
