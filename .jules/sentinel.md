## 2024-05-24 - Stop pprof and expvar exposure on public APIs
**Vulnerability:** Default mux endpoints like `/debug/pprof/*` and `/debug/vars` were inadvertently exposed on the public internet by importing `_ "net/http/pprof"` and `_ "expvar"` in the public POSIX API server. This exposes sensitive memory, metrics, and profiling data (CWE-200).
**Learning:** Importing these packages registers their endpoints globally on `http.DefaultServeMux`. If `http.ListenAndServe()` is called with `nil` as the handler, it serves `http.DefaultServeMux`, which exposes these endpoints to the internet if not restricted by a firewall or bound to a private IP. Even though the application uses `http.Server` and binds `logHandler` explicitly to `"/"` via `http.Handle("/", logHandler)`, because `logHandler` does not implement all URLs and `/debug` is not shielded, any unhandled endpoint falls back or could potentially be mapped. Wait, the code calls `http.Handle("/", logHandler)`, which maps `/` on the default ServeMux. Because `/` is a catch-all, requests to `/debug/pprof/` would normally go to the pprof handlers which register themselves with exact paths like `/debug/pprof/` and `/debug/pprof/cmdline` on the `http.DefaultServeMux`. These exact paths take precedence over the `/` catch-all! Therefore, they are exposed!
**Prevention:** Avoid blanket importing `net/http/pprof` or `expvar` in binaries exposed directly to untrusted networks. If profiling or debug metrics are required, register them explicitly on a separate, internal HTTP server or mux listening only on localhost (`127.0.0.1:port`).
