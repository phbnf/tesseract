## 2026-05-31 - Fix anonymous profiling endpoint exposure (CWE-200)
**Vulnerability:** Profiling and metric endpoints (`/debug/pprof/*` and `/debug/vars`) were inadvertently exposed on the global `http.DefaultServeMux` due to anonymous imports (`_ "net/http/pprof"` and `_ "expvar"`) in the GCP and POSIX cloud personality entry points (`cmd/tesseract/gcp/main.go` and `cmd/tesseract/posix/main.go`).
**Learning:** In Go applications that configure their own HTTP multiplexers, using anonymous imports for `net/http/pprof` or `expvar` automatically registers their handlers on `http.DefaultServeMux`. If `http.ListenAndServe` or similar functions are called with `nil` as the handler, or if the default mux is otherwise exposed to external traffic, this creates a CWE-200 vulnerability by leaking internal application state, performance metrics, and memory profiles.
**Prevention:** Do not use anonymous imports for `net/http/pprof` or `expvar` in production entry points. If profiling or metrics are required, explicitly register them on a private, internal-only multiplexer or an unexposed port, separate from the public-facing API routes. Use OpenTelemetry for observability instead of `expvar` where applicable.
