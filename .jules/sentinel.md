## 2024-05-15 - [CRITICAL] Removed pprof endpoint exposure on DefaultServeMux
**Vulnerability:** The POSIX cloud personality binary (`cmd/tesseract/posix/main.go`) included a blank import of `_ "net/http/pprof"`. This automatically registers `net/http/pprof` handlers to `http.DefaultServeMux`. Although the application runs on its own `ServeMux`, the exposure of the default `ServeMux` presents a severe CWE-200 (Exposure of Sensitive Information) risk. Attackers could access `/debug/pprof/*` endpoints to profile the application, view stack traces, memory allocations (which might leak sensitive data), or cause a Denial of Service via intensive profiling operations.
**Learning:** In Go applications, especially those exposing public-facing HTTP servers, avoid blank imports of `net/http/pprof` unless strictly controlled and bound to a private/admin interface. `pprof` handlers blindly attach to `http.DefaultServeMux`, leading to unintentional exposure if the default mux is ever served or exposed indirectly.
**Prevention:** Never use the blank import `_ "net/http/pprof"` in production application entry points. If profiling is necessary, explicitly register pprof handlers to a dedicated, internal-only `http.ServeMux` that is not exposed to the public internet. Ensure static analysis tools like `gosec` are configured to catch `G108` (Profiling endpoint exposed).
