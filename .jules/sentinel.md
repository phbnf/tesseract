## 2025-05-18 - [Exposed Profiling Endpoint (CWE-200)]
**Vulnerability:** The profiling endpoint `/debug/pprof` was inadvertently exposed on the public HTTP server in the GCP and POSIX entry points (`cmd/tesseract/gcp/main.go` and `cmd/tesseract/posix/main.go`) because of the `_ "net/http/pprof"` import combined with the use of `http.DefaultServeMux`.
**Learning:** Importing `net/http/pprof` for side-effects automatically registers profiling endpoints to `http.DefaultServeMux`. If `http.ListenAndServe` or `http.Server.ListenAndServe` is called without specifying a custom multiplexer (or if handlers are mapped onto the default one via `http.Handle`), these sensitive endpoints become publicly accessible, leading to a CWE-200 Information Exposure vulnerability.
**Prevention:** Avoid blanket importing `net/http/pprof` in production-facing HTTP server binaries, especially when routing is handled by `http.DefaultServeMux`. If profiling is needed, register pprof handlers explicitly to an internal/admin-only multiplexer on a separate port or isolated network path, independent of the public API surface.
