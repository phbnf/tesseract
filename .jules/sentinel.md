## 2025-07-02 - DefaultServeMux Exposure via Blank Imports
**Vulnerability:** The cloud personalities for GCP and POSIX (`cmd/tesseract/gcp/main.go` and `cmd/tesseract/posix/main.go`) included blank imports for `net/http/pprof` and/or `expvar`. These imports automatically register handlers on `http.DefaultServeMux`. If `http.ListenAndServe` or similar is called using `DefaultServeMux`, it exposes internal metrics and profiling endpoints (`/debug/pprof/*` and `/debug/vars`) publicly without authentication, leading to CWE-200 (Information Exposure).
**Learning:** In cloud entry points and publicly exposed servers, using blank imports for debugging or metrics packages inadvertently attaches them to the global mux, ignoring custom multiplexers or routers that might have been intended for actual application traffic.
**Prevention:** Avoid blank imports of `net/http/pprof` and `expvar` in production binaries. If profiling or debugging metrics are required, explicitly attach them to an internal, authenticated, or separate administrative `http.ServeMux` that is not exposed to the public internet. Use explicit OpenTelemetry (otel) instead.
