## 2024-05-01 - Global /debug/pprof and /debug/vars Exposure in External Services
**Vulnerability:** The HTTP server entry points in `cmd/tesseract/gcp/main.go` and `cmd/tesseract/posix/main.go` inadvertently expose `net/http/pprof` endpoints (`/debug/pprof/*`) and `expvar` (`/debug/vars`) on the `http.DefaultServeMux`, which is served by default.
**Learning:** Importing `net/http/pprof` or `expvar` for profiling or metrics in a global context can leak sensitive application state, internal performance details, and potentially source code paths if exposed on an internet-facing endpoint. This is particularly problematic in cloud environments.
**Prevention:** Avoid blank imports of `_ "net/http/pprof"` and `_ "expvar"` in production binaries. If profiling or metrics are required, explicitly bind them to an internal, authenticated, or separate administrative `http.ServeMux` rather than `http.DefaultServeMux`. Use OpenTelemetry or structured logging for production observability.
